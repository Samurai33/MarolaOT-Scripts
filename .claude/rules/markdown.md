# Markdown and documentation

- Write in Portuguese unless the existing document or external contract is English.
- Keep claims traceable to repository evidence or cited primary sources.
- Use exact file paths, commands, maturity states, and dates.
- Do not paste long copyrighted guides; summarize and link.
- Prefer relative links for repository files.
- Keep headings hierarchical and tables small enough to read on mobile.
- Mark examples, provisional data, assumptions, and unvalidated coordinates explicitly.
- Do not state that tests, CI, client validation, or gameplay validation happened without evidence.
- Update the catalog, changelog, package README, and manifest together when status changes.
- Avoid duplicated procedures; link to the canonical skill or operating document.
