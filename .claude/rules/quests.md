# Quest and access packages

- Treat editorial guides as navigation references, not executable truth.
- Cross-check quest progression with primary server data when available.
- Every coordinate begins as provisional until confirmed in MarolaOT.
- Preserve both wiki/global and MarolaOT coordinates when they differ.
- Split long quests by mission and by reversible segment.
- Stop before NPC dialogue, item use, storage-changing object, lever, boss, arena, prison/capture state, multiplayer gate, rare-item consumption, reward choice, or irreversible action.
- Do not create continuous unattended quest scripts.
- Keep M0–M2 packages non-executable and enforce that in CI.
- A segment requires a safe origin, visible manual stop, recovery/return path, and dry-run evidence.
- Quest completion and unlock claims require Quest Log or server evidence.
