# Política de segurança

## Dados que nunca devem ser publicados

- senhas de contas ou personagens;
- hashes de senha;
- tokens GitHub, SSH, Tailscale ou Cloudflare;
- credenciais do banco de dados;
- IPs privados ou públicos que não precisem ser documentados;
- backups de `profile_1.json` contendo dados pessoais;
- dumps do banco de dados do servidor.

## Reporte de vulnerabilidades

Não abra uma issue pública contendo credenciais ou detalhes exploráveis. Envie o relato de forma privada ao mantenedor do projeto.

## Execução segura

Revise o script, mantenha backups e execute alterações do vBot somente com o MarolaOT fechado.
